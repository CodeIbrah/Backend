# 后端模板

**企业级后端，采用混合架构（单体 + 微服务）**

使用NestJS和Express构建的生产级后端模板，采用混合架构，将功能丰富的单体应用与独立的微服务结合。包含全面的可观测性、AI驱动的错误诊断、分析、基于队列的处理和自动化的事件响应。

## 架构概述

该系统采用混合架构，将功能丰富的单体应用与独立的微服务结合：

`+-----------------------------------------------------------------------------------------+
|                              混合架构                                |
+-----------------------------------------------------------------------------------------+
|                                                                                         |
|  +-----------------------------------------------------------------------------+       |
|  |                                主单体应用                              |       |
|  |                                                                                     |       |
|  |  +-----------------------------------------------------------------+           |       |
|  |  |                        核心域服务                              |           |       |
|  |  |  +-------------+  +-------------+  +-------------+  +-------------+  |           |       |
|  |  |  |  认证        |  |  用户        |  |  产品        |  |  订单        |  |           |       |
|  |  |  +-------------+  +-------------+  +-------------+  +-------------+  |           |       |
|  |  +-----------------------------------------------------------------+           |       |
|  |                                                                                     |       |
|  |  +-----------------------------------------------------------------+           |       |
|  |  |                        支持服务                                |           |       |
|  |  |  +-------------+  +-------------+  +-------------+  +-------------+  |           |       |
|  |  |  |  日志        |  |  遥测        |  |  队列        |  |  报告        |  |           |       |
|  |  |  +-------------+  +-------------+  +-------------+  +-------------+  |           |       |
|  |  +-----------------------------------------------------------------+           |       |
|  +-----------------------------------------------------------------------------+       |
|                                                                                         |
|  +-----------------------------------------------------------------------------+       |
|  |                                微服务                                |       |
|  |                                                                                     |       |
|  |  +-----------------------------------------------------------------+           |       |
|  |  |                        独立服务                                |           |       |
|  |  |  +-------------+  +-------------+  +-------------+  +-------------+  |           |       |
|  |  |  |  认证        |  |  通知        |  |  支付        |  |  库存        |  |           |       |
|  |  |  +-------------+  +-------------+  +-------------+  +-------------+  |           |       |
|  |  +-----------------------------------------------------------------+           |       |
|  +-----------------------------------------------------------------------------+       |
|                                                                                         |
+-----------------------------------------------------------------------------------------+`

## 技术栈

- **后端**: NestJS/Express (Node.js/TypeScript)
- **数据库**: PostgreSQL + Prisma
- **缓存**: Redis
- **队列系统**: BullMQ
- **日志**: Winston
- **追踪**: OpenTelemetry + Jaeger
- **指标**: Prometheus
- **日志聚合**: Loki
- **仪表盘**: Grafana
- **AI系统**: 用于错误诊断和事件响应的自定义AI代理

## 项目结构

`后端模板/
+-- main/                  # 主单体应用
+-- microservices/         # 独立微服务
|   +-- auth-service/       # 认证服务
|   +-- notifications-service/ # 通知服务
|   +-- payment-service/    # 支付处理服务
|   +-- users-service/      # 用户管理服务
+-- gateway/               # API网关
+-- infrastructure/        # 基础设施配置
+-- scripts/               # 实用脚本
+-- reports/               # 自动报告
+-- skills/                # AI代理技能
+-- context/               # 上下文信息
+-- docs/                  # 文档
+-- .env.example           # 环境变量模板
+-- package.json           # 项目依赖
+-- README.md              # 项目文档`

## 快速开始

1. 克隆仓库
2. 安装依赖:

npm install 3. 设置环境变量 (复制 .env.example 到 .env) 4. 启动开发服务器:
npm run dev

## 开发环境设置

1. 安装Node.js (v20+)
2. 安装PostgreSQL
3. 安装Redis
4. 安装Docker (用于基础设施)

## 环境变量

基于 .env.example 创建 .env 文件并配置：

`

# 数据库

DATABASE_URL=postgresql://用户:密码@localhost:5432/数据库名称

# Redis

REDIS_URL=redis://localhost:6379

# JWT

JWT_SECRET=你的jwt密钥

# Stripe

STRIPE_SECRET_KEY=你的stripe密钥
STRIPE_WEBHOOK_SECRET=你的webhook密钥

# 可观测性

PROMETHEUS_URL=http://localhost:9090
GRAFANA_URL=http://localhost:3000
LOKI_URL=http://localhost:3100
JAEGER_URL=http://localhost:16686

# AI系统

AI_DOCTOR_API_KEY=你的ai_doctor_api密钥
`

## AI错误诊断系统

AI错误诊断系统提供自动化的错误诊断和事件响应：

- **错误分析**: AI驱动的错误分析和根本原因识别
- **事件响应**: 自动化的事件响应和解决
- **错误模式**: 重复错误模式的检测
- **建议**: 错误解决的可执行建议

## 分析系统

分析系统提供系统性能和用户行为的全面洞察：

- **用户分析**: 用户活动和行为的跟踪
- **性能分析**: 系统性能指标的监控
- **错误分析**: 错误模式和趋势的分析
- **业务分析**: 业务指标和KPI的洞察

## 队列系统

队列系统处理异步处理和后台作业：

- **作业处理**: 后台作业的处理
- **作业调度**: 定期作业的调度
- **作业监控**: 作业状态和进度的监控

## 警报系统

警报系统提供实时通知和警报：

## 自动报告

## 故障排除

常见问题和解决方案：

- **问题**: 应用程序未启动
  **解决方案**: 检查日志并确保所有依赖项都已安装

- **问题**: 数据库连接错误
  **解决方案**: 验证数据库凭据和连接URL

- **问题**: API端点未工作
  **解决方案**: 检查API网关配置和微服务状态
