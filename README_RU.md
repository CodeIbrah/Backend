# Шаблон Backend

**Корпоративный Backend с гибридной архитектурой (Монолит + Микросервисы)**

Производственный шаблон backend, построенный на NestJS и Express, сочетающий в себе функциональный монолит с независимыми микросервисами. Включает всестороннюю наблюдаемость, диагностику ошибок с помощью ИИ, аналитику, обработку задач через очереди и автоматический ответ на инциденты.

## Технологический стек

- **Backend**: NestJS/Express (Node.js/TypeScript)
- **База данных**: PostgreSQL + Prisma
- **Кэширование**: Redis
- **Система очередей**: BullMQ
- **Логирование**: Winston
- **Трассировка**: OpenTelemetry + Jaeger
- **Метрики**: Prometheus
- **Агрегация логов**: Loki
- **Дашборд**: Grafana
- **Система ИИ**: Кастомные агенты ИИ для диагностики ошибок и реагирования на инциденты

## Быстрый старт

1. Клонируйте репозиторий
2. Установите зависимости:

npm install 3. Настройте переменные окружения (скопируйте .env.example в .env) 4. Запустите сервер разработки:
npm run dev

## Настройка среды разработки

1. Установите Node.js (v20+)
2. Установите PostgreSQL
3. Установите Redis
4. Установите Docker (для инфраструктуры)

## Переменные окружения

Создайте файл .env на основе .env.example и настройте:

`

# База данных

DATABASE_URL=postgresql://пользователь:пароль@localhost:5432/названиебазы

# Redis

REDIS_URL=redis://localhost:6379

# JWT

JWT*SECRET=ваш_jwt*секрет

# Stripe

STRIPE*SECRET_KEY=ваш_stripe*секретный*ключ
STRIPE_WEBHOOK_SECRET=ваш_webhook*секрет

# Наблюдаемость

PROMETHEUS_URL=http://localhost:9090
GRAFANA_URL=http://localhost:3000
LOKI_URL=http://localhost:3100
JAEGER_URL=http://localhost:16686

# Система ИИ

AI*DOCTOR_API_KEY=ваш_ai_doctor_api*ключ
`

## API Endpoints

Система предоставляет следующие основные endpoints:

- **Сервис аутентификации**: /api/auth
- **Сервис пользователей**: /api/users
- **Сервис платежей**: /api/payments
- **Сервис уведомлений**: /api/notifications
- **Сервис инвентаризации**: /api/inventory

## Наблюдаемость

Система включает всестороннюю наблюдаемость:

- **Метрики**: Prometheus для сбора метрик
- **Логи**: Winston + Loki для агрегации логов
- **Трассировка**: OpenTelemetry + Jaeger для распределенной трассировки
- **Дашборды**: Grafana для визуализации

## Система AI Error Doctor

Система AI Error Doctor предоставляет автоматизированную диагностику ошибок и реагирование на инциденты:

- **Анализ ошибок**: Анализ ошибок с помощью ИИ и идентификация коренных причин
- **Реакция на инциденты**: Автоматическое реагирование и решение инцидентов
- **Шаблоны ошибок**: Обнаружение повторяющихся шаблонов ошибок
- **Рекомендации**: Действенные рекомендации по устранению ошибок

## Аналитическая система

Аналитическая система предоставляет всесторонние данные о производительности системы и поведении пользователей:

- **Анализ пользователей**: Отслеживание активности и поведения пользователей
- **Анализ производительности**: Мониторинг метрик производительности системы
- **Анализ ошибок**: Анализ шаблонов и тенденций ошибок
- **Анализ бизнеса**: Инсайты по бизнес-метрикам и KPI

## Система очередей

Система очередей обрабатывает асинхронную обработку и фоновые задачи:

- **Обработка задач**: Обработка фоновых задач
- **Планирование задач**: Планирование повторяющихся задач
- **Мониторинг задач**: Мониторинг статуса и прогресса задач

## Система оповещений

Система оповещений предоставляет уведомления и оповещения в реальном времени:

- **Типы оповещений**: Различные типы оповещений (ошибка, предупреждение, информация)

## Устранение неполадок

Распространенные проблемы и решения:

- **Проблема**: Приложение не запускается
  **Решение**: Проверьте логи и убедитесь, что все зависимости установлены

- **Проблема**: Ошибки подключения к базе данных
  **Решение**: Проверьте учетные данные базы данных и URL подключения

- **Проблема**: API endpoint не работает
  **Решение**: Проверьте конфигурацию API Gateway и статус микросервиса
