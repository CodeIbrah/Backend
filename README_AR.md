# قالب الخلفية

**الخلفية للشركات مع معمارية هجينة (المنحى + الخدمات الصغيرة)**

قالب خلفية إنتاجية مبنية على NestJS و Express، حيث يتم دمج منحى مع ميزات متقدمة مع خدمات صغيرة مستقلة. يتضمن مراقبة شاملة، تشخيص الأخطاء المدعوم بالذكاء الاصطناعي، التحليل، المعالجة القائمة على الطابور، والاستجابة التلقائية للحوادث.

## مجموعة التقنيات

- **الخلفية**: NestJS/Express (Node.js/TypeScript)
- **قاعدة البيانات**: PostgreSQL + Prisma
- **التخزين المؤقت**: Redis
- **نظام الطابور**: BullMQ
- **التسجيل**: Winston
- **التتبع**: OpenTelemetry + Jaeger
- **المقاييس**: Prometheus
- **تجميع السجلات**: Loki
- **لوحة التحكم**: Grafana
- **نظام الذكاء الاصطناعي**: وكلاء ذكاء اصطناعي مخصصة لتشخيص الأخطاء والاستجابة للحوادث

## هيكل المشروع

`قالب-الخلفية/
+-- main/                  # التطبيق المنحى الرئيسي
+-- microservices/         # الخدمات الصغيرة المستقلة
|   +-- auth-service/       # خدمة المصادقة
|   +-- notifications-service/ # خدمة الإشعارات
|   +-- payment-service/    # خدمة معالجة المدفوعات
|   +-- users-service/      # خدمة إدارة المستخدمين
+-- gateway/               # بوابة API

+-- scripts/               # نصوص المساعدة
+-- reports/               # التقارير التلقائية
+-- skills/                # مهارات وكلاء الذكاء الاصطناعي
+-- context/               # المعلومات السياقية
+-- docs/                  # الوثائق
+-- .env.example           # نموذج متغيرات البيئة
+-- package.json           # تبعيات المشروع
+-- README.md              # وثائق المشروع`

## البدء السريع

1. استنسخ المستودع
2. تثبيت التبعيات:

npm install 3. إعداد متغيرات البيئة (انسخ .env.example إلى .env) 4. تشغيل خادم التطوير:
npm run dev

## إعداد بيئة التطوير

1. تثبيت Node.js (v20+)
2. تثبيت PostgreSQL
3. تثبيت Redis
4. تثبيت Docker (للبنية التحتية)

## متغيرات البيئة

أنشئ ملف .env استنادًا إلى .env.example وقم بتكوينه:

`

# قاعدة البيانات

DATABASE*URL=postgresql://المستخدم:كلمة*المرور@localhost:5432/اسم_القاعدة

# Redis

REDIS_URL=redis://localhost:6379

# JWT

JWT*SECRET=مفتاح_jwt*الخاص_بك

# Stripe

STRIPE*SECRET_KEY=مفتاح_stripe*الخاص*بك
STRIPE_WEBHOOK_SECRET=مفتاح_webhook*الخاص_بك

# المراقبة

PROMETHEUS_URL=http://localhost:9090
GRAFANA_URL=http://localhost:3000
LOKI_URL=http://localhost:3100
JAEGER_URL=http://localhost:16686

# نظام الذكاء الاصطناعي

AI*DOCTOR_API_KEY=مفتاح_ai_doctor_api*الخاص_بك
`

## نقاط النهاية API

يقدم النظام النقاط النهائية الرئيسية التالية:

- **خدمة المصادقة**: /api/auth
- **خدمة المستخدمين**: /api/users
- **خدمة المدفوعات**: /api/payments
- **خدمة الإشعارات**: /api/notifications
- **خدمة المخزون**: /api/inventory

## المراقبة

يضم النظام ميزات مراقبة شاملة:

- **المقاييس**: Prometheus لجمع المقاييس
- **السجلات**: Winston + Loki لتجميع السجلات
- **التتبع**: OpenTelemetry + Jaeger للتتبع الموزع
- **لوحات التحكم**: Grafana للتصور

## نظام AI Error Doctor

يقدم نظام AI Error Doctor تشخيصًا واستجابة تلقائيتين للأخطاء والحوادث:

- **تحليل الأخطاء**: تحليل الأخطاء المدعوم بالذكاء الاصطناعي وتحديد الأسباب الجذرية
- **استجابة الحوادث**: استجابة تلقائية وحل الحوادث
- **أنماط الأخطاء**: اكتشاف أنماط الأخطاء المتكررة
- **التوصيات**: توصيات قابلة للتنفيذ لحل الأخطاء

## نظام التحليل

يقدم نظام التحليل رؤى شاملة حول أداء النظام وسلوك المستخدم:

- **تحليل المستخدمين**: تتبع نشاط وسلوك المستخدمين
- **تحليل الأداء**: مراقبة مقاييس أداء النظام
- **تحليل الأخطاء**: تحليل أنماط وتوجهات الأخطاء
- **تحليل الأعمال**: رؤى حول مقاييس وأهداف الأداء (KPI) للأعمال

## نظام الطابور

ينظم نظام الطابور المعالجة غير المتزامنة والوظائف الخلفية:

- **معالجة المهام**: معالجة المهام الخلفية
- **جدولة المهام**: جدولة المهام المتكررة
- **مراقبة المهام**: مراقبة حالة وتقدم المهام

## نظام التنبيهات

يقدم نظام التنبيهات إشعارات وتذكيرات في الوقت الفعلي:

- **أنواع التنبيهات**: أنواع مختلفة من التنبيهات (خطأ، تحذير، معلومات)
- **قنوات التنبيهات**: قنوات متعددة للتنبيهات (البريد الإلكتروني، Slack، PagerDuty)
- **تسلسخ التنبيهات**: سياسات تسلسخ التنبيهات

## التقارير التلقائية

ينشئ النظام تقارير تلقائية لمراقبة والتحليل:

- **تقارير الأداء**: تقارير حول أداء النظام
- **تقارير الأخطاء**: تقارير حول أنماط وتوجهات الأخطاء
- **تقارير الأعمال**: تقارير حول مقاييس وأهداف الأداء (KPI) للأعمال

## الأمان

يضم النظام ميزات أمان شاملة:

- **المصادقة**: مصادقة مبنية على JWT
- **التحقيق**: التحكم في الوصول على أساس الأدوار
- **حماية البيانات**: تشفير البيانات الحساسة
- **تدقيق الأمان**: تدقيقات أمنية منتظمة ووسائل فحص الضعفيات

## الاختبارات

يضم النظام اختبارات شاملة:

- **اختبارات الوحدة**: اختبارات الوحدة للمكونات الفردية
- **اختبارات التكامل**: اختبارات التكامل للتفاعل بين المكونات
- **اختبارات نهاية إلى نهاية**: اختبارات نهاية إلى نهاية لسلاسل المستخدم
- **اختبارات التحميل**: اختبارات التحميل لتقييم الأداء

## إعداد Docker

يضم النظام تكوين Docker للإنشاء السهل:

- **Docker Compose**: تكوين متعدد الحاويات Docker
- **صور Docker**: صور Docker مسبقة الإنشاء
- **Docker Hub**: صور Docker متاحة على Docker Hub

## إصلاح الأخطاء

المشاكل الشائعة وحلولها:

- **المشكلة**: التطبيق لا يبدأ
  **الحل**: تحقق من السجلات وتأكد من تثبيت جميع التبعيات

- **المشكلة**: أخطاء اتصال قاعدة البيانات
  **الحل**: تحقق من بيانات اعتماد قاعدة البيانات وURL الاتصال

- **المشكلة**: نقطة النهاية API لا تعمل
  **الحل**: تحقق من تكوين بوابة API وحالة الخدمة الصغيرة
