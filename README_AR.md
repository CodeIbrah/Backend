**الخلفية**: NestJS/Express (Node.js/TypeScript)\n- **قاعدة البيانات**: PostgreSQL + Prisma\n- **التخزين المؤقت**: Redis\n- **نظام الطابور**: BullMQ\n- **التسجيل**: Winston\n- **التتبع**: OpenTelemetry + Jaeger\n- **المقاييس**: Prometheus\n- **تجميع السجلات**: Loki\n- **لوحة التحكم**: Grafana\n- **نظام الذكاء الاصطناعي**: وكلاء ذكاء اصطناعي مخصصة لتشخيص الأخطاء والاستجابة للحوادث\n\n## هيكل المشروع\n\n`\nقالب-الخلفية/\n├── main/                  # التطبيق المنحى الرئيسي\n├── microservices/         # الخدمات الصغيرة المستقلة\n│   ├── auth-service/       # خدمة المصادقة\n│   ├── notifications-service/ # خدمة الإشعارات\n│   ├── payment-service/    # خدمة معالجة المدفوعات\n│   └── users-service/      # خدمة إدارة المستخدمين\n├── gateway/               # بوابة API\n├── infrastructure/        # تكوين البنية التحتية\n├── scripts/               # نصوص المساعدة\n├── reports/               # التقارير التلقائية\n├── skills/                # مهارات وكلاء الذكاء الاصطناعي\n├── context/               # المعلومات السياقية\n├── docs/                  # الوثائق\n├── .env.example           # نموذج متغيرات البيئة\n├── package.json           # تبعيات المشروع\n└── README.md              # وثائق المشروع\n`\n\n## البدء السريع\n\n1. استنسخ المستودع\n2. تثبيت التبعيات:

pm install\n3. إعداد متغيرات البيئة (انسخ .env.example إلى .env)\n4. تشغيل خادم التطوير:
pm run dev\n\n## إعداد بيئة التطوير\n\n1. تثبيت Node.js (v20+)\n2. تثبيت PostgreSQL\n3. تثبيت Redis\n4. تثبيت Docker (للبنية التحتية)\n\n## متغيرات البيئة\n\nأنشئ ملف .env استنادًا إلى .env.example وقم بتكوينه:\n\n`\n# قاعدة البيانات\nDATABASE_URL=postgresql://المستخدم:كلمة_المرور@localhost:5432/اسم_القاعدة\n\n# Redis\nREDIS_URL=redis://localhost:6379\n\n# JWT\nJWT_SECRET=مفتاح_jwt_الخاص_بك\n\n# Stripe\nSTRIPE_SECRET_KEY=مفتاح_stripe_الخاص_بك\nSTRIPE_WEBHOOK_SECRET=مفتاح_webhook_الخاص_بك\n\n# المراقبة\nPROMETHEUS_URL=http://localhost:9090\nGRAFANA_URL=http://localhost:3000\nLOKI_URL=http://localhost:3100\nJAEGER_URL=http://localhost:16686\n\n# نظام الذكاء الاصطناعي\nAI_DOCTOR_API_KEY=مفتاح_ai_doctor_api_الخاص_بك\n`\n\n## نقاط النهاية API\n\nيقدم النظام النقاط النهائية الرئيسية التالية:\n\n- **خدمة المصادقة**: /api/auth\n- **خدمة المستخدمين**: /api/users\n- **خدمة المدفوعات**: /api/payments\n- **خدمة الإشعارات**: /api/notifications\n- **خدمة المخزون**: /api/inventory\n\n## المراقبة\n\nيضم النظام ميزات مراقبة شاملة:\n\n- **المقاييس**: Prometheus لجمع المقاييس\n- **السجلات**: Winston + Loki لتجميع السجلات\n- **التتبع**: OpenTelemetry + Jaeger للتتبع الموزع\n- **لوحات التحكم**: Grafana للتصور\n\n## نظام AI Error Doctor\n\nيقدم نظام AI Error Doctor تشخيصًا واستجابة تلقائيتين للأخطاء والحوادث:\n\n- **تحليل الأخطاء**: تحليل الأخطاء المدعوم بالذكاء الاصطناعي وتحديد الأسباب الجذرية\n- **استجابة الحوادث**: استجابة تلقائية وحل الحوادث\n- **أنماط الأخطاء**: اكتشاف أنماط الأخطاء المتكررة\n- **التوصيات**: توصيات قابلة للتنفيذ لحل الأخطاء\n\n## نظام التحليل\n\nيقدم نظام التحليل رؤى شاملة حول أداء النظام وسلوك المستخدم:\n\n- **تحليل المستخدمين**: تتبع نشاط وسلوك المستخدمين\n- **تحليل الأداء**: مراقبة مقاييس أداء النظام\n- **تحليل الأخطاء**: تحليل أنماط وتوجهات الأخطاء\n- **تحليل الأعمال**: رؤى حول مقاييس وأهداف الأداء (KPI) للأعمال\n\n## نظام الطابور\n\nينظم نظام الطابور المعالجة غير المتزامنة والوظائف الخلفية:\n\n- **معالجة المهام**: معالجة المهام الخلفية\n- **جدولة المهام**: جدولة المهام المتكررة\n- **مراقبة المهام**: مراقبة حالة وتقدم المهام\n\n## نظام التنبيهات\n\nيقدم نظام التنبيهات إشعارات وتذكيرات في الوقت الفعلي:\n\n- **أنواع التنبيهات**: أنواع مختلفة من التنبيهات (خطأ، تحذير، معلومات)\n- **قنوات التنبيهات**: قنوات متعددة للتنبيهات (البريد الإلكتروني، Slack، PagerDuty)\n- **تسلسخ التنبيهات**: سياسات تسلسخ التنبيهات\n\n## التقارير التلقائية\n\nينشئ النظام تقارير تلقائية لمراقبة والتحليل:\n\n- **تقارير الأداء**: تقارير حول أداء النظام\n- **تقارير الأخطاء**: تقارير حول أنماط وتوجهات الأخطاء\n- **تقارير الأعمال**: تقارير حول مقاييس وأهداف الأداء (KPI) للأعمال\n\n## الأمان\n\nيضم النظام ميزات أمان شاملة:\n\n- **المصادقة**: مصادقة مبنية على JWT\n- **التحقيق**: التحكم في الوصول على أساس الأدوار\n- **حماية البيانات**: تشفير البيانات الحساسة\n- **تدقيق الأمان**: تدقيقات أمنية منتظمة ووسائل فحص الضعفيات\n\n## الاختبارات\n\nيضم النظام اختبارات شاملة:\n\n- **اختبارات الوحدة**: اختبارات الوحدة للمكونات الفردية\n- **اختبارات التكامل**: اختبارات التكامل للتفاعل بين المكونات\n- **اختبارات نهاية إلى نهاية**: اختبارات نهاية إلى نهاية لسلاسل المستخدم\n- **اختبارات التحميل**: اختبارات التحميل لتقييم الأداء\n\n## إعداد Docker\n\nيضم النظام تكوين Docker للإنشاء السهل:\n\n- **Docker Compose**: تكوين متعدد الحاويات Docker\n- **صور Docker**: صور Docker مسبقة الإنشاء\n- **Docker Hub**: صور Docker متاحة على Docker Hub\n\n## إصلاح الأخطاء\n\nالمشاكل الشائعة وحلولها:\n\n- **المشكلة**: التطبيق لا يبدأ\n **الحل**: تحقق من السجلات وتأكد من تثبيت جميع التبعيات\n\n- **المشكلة**: أخطاء اتصال قاعدة البيانات\n **الحل**: تحقق من بيانات اعتماد قاعدة البيانات وURL الاتصال\n\n- **المشكلة**: نقطة النهاية API لا تعمل\n **الحل**: تحقق من تكوين بوابة API وحالة الخدمة الصغيرة\n\n## المساهمة\n\nانظر [CONTRIBUTING.md](CONTRIBUTING.md) لخطوط الإرشاد للمساهمة.\n\n## الرخصة\n\nهذا المشروع مرخص بموجب رخصة MIT -
